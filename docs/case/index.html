Work backwards from €8M

Covers the €450k/mo requirement with ~15% buffer. Every assumption becomes a dashboard metric, so the model self-corrects from month 3. Zero additional headcount, which is the PE constraint and, not coincidentally, the SalesPlaybook operating model.

02 · The Signal Thesis

Every DACH ECC customer is in a forced migration window

SAP ECC mainstream maintenance ends 2027 (extended 2030). Archiving before migration cuts the HANA footprint and migration cost; decommissioning after is a mandatory workstream. That is a detectable, budgeted, deadline-driven "why now" for exactly ArchivoX's offer, and it became Signal S1.

05 · Beyond the Cog

The full engine is five layers

The built component is layers 1 to 4 in miniature for one signal type. The engagement scales it across the taxonomy without changing the architecture.

L5 · Feedback

Signal→meeting→opp conversion per signal type. Kill weak signals, scale strong ones. CRO dashboard: pipeline by signal, cost per SQL, SDR time saved.

L4 · Activation

Message-based: Tier A human-approved 1:1 email + LinkedIn, Tier B automated sequences on warmed domains, Tier C nurture. Phone stays an optional client-side lever for ArchivoX's existing SDRs.

L3 · Scoring & Routing

Fit × signal (type weight, recency decay, evidence confidence) → tier, owner, SLA. Suppression reads both CRMs so no customer or open opp is ever touched: the only hard cross-CRM requirement, and it is read-only, therefore cheap. Writes go to one CRM: HubSpot, already in ArchivoX's stack.

L2 · Signal Detection

Code watchers for German sources no tool covers (job boards, Bundesanzeiger, registers) plus Dealfront/Echobot for DACH-native triggers and web intent. Everything normalizes to one signal_event shape.

L1 · Account Universe

The full DACH TAM (~5k accounts) with fit scores, tech stack and persisted contacts. Built from ArchivoX's own CRMs first, then Dealfront. The system of intelligence between market and CRM: signals and people are reused, never re-researched.

Deliberately not built: no CRM merge (flagged to the CRO as a separate workstream), no custom sequencing, no UI beyond the cockpit. The engine is the brain between market and CRM, nothing more. In DACH, legal reality shapes the motion rather than footnoting it: UWG §7 pushes Tier A toward individual, evidence-anchored 1:1 messages at low volume, which is exactly the motion that converts in enterprise anyway.
